# Chat Trigger Rules

这个文件用于把 `file-organizer/` 的聊天触发行为写死，减少每次临场判断。

## 目标

让“用户发链接让我记录”这件事尽量变成标准操作，而不是临时分析题。

## 默认触发模式

满足以下条件时，默认直接进入 ingest：

1. 用户消息里包含一个 URL
2. 同时包含记录类意图，例如：
   - 记录这个
   - 收一下
   - 整理一下
   - 存一下
   - 归档这个
   - 记一下

## 默认执行命令

```bash
python3 file-organizer/scripts/ingest_any_url.py "<url>"
```

如果用户同时表达“顺手出个综述 / digest / 摘要”，则可用：

```bash
python3 file-organizer/scripts/ingest_any_url.py "<url>" --with-digests
```

## 决策规则

- 优先走统一入口，不要先手动挑 wechat / web 子脚本
- 只有统一入口失败时，才降级到底层脚本排查
- 成功后默认向用户回报：来源形态、完整度、产出文件、关键要点、待验证项
- 如果用户只发链接但上下文已经明显处于“帮我归档材料”的流程，也默认可直接 ingest

## 不自动触发的情况

以下情况不要直接入库：
- 用户只是在聊天里转发一个链接讨论，并没有记录意图
- 用户明确说“先别整理”“只看看”“先读一下再说”
- 链接只是问题背景，而不是要入知识库的对象

## 一句话原则

对 file-organizer 来说，“链接 + 记录类动词” 就等于 “执行统一 ingest”。
