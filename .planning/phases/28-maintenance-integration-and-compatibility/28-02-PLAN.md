# Plan 28-02: Add Artifact-Health Findings For Views, Agent Exports, And Validation

<objective>
Extend the maintenance findings layer so the existing maintenance loop can queue actionable tasks for v0.2 validation records, generated wiki page drift, human/source view tasks, and agent export refreshes without directly mutating derived artifacts.
</objective>

<data_flow>
1. Plan 28-01 creates a `targets` document from existing v0.2 artifacts.
2. `build_findings(graph, targets=targets)` keeps graph findings and appends artifact-health findings.
3. Validation report records become `object_validation_record` findings with precise object/path targets.
4. Wikiization drift tasks become generated page repair/source traceability findings.
5. View task queue entries become view regeneration/review findings with `regeneration_command: wikify views`.
6. Missing or stale agent export artifacts become agent export refresh findings with `regeneration_command: wikify agent export`.
7. Planner and task queue map these findings to queued semantic actions; `run-task` can hand them to agents, but derived views/agent exports instruct regeneration instead of direct patching.
</data_flow>

<threat_model>
## Threat Model

**Assets:** validation records, generated page metadata, view tasks, agent export artifacts, task queue instructions.

**Threats and mitigations:**
- Derived artifact divergence: view and agent export tasks carry regeneration commands instead of patch instructions.
- User interruption creep: findings are queued with `requires_user: False` unless existing task evidence explicitly requires manual review.
- Raw source reanalysis: artifact-health findings read only `.wikify/`, `artifacts/`, `views/`, `wiki/`, and `graph/` artifacts.
- Action risk mismatch: planner marks new actions semantic/generated-content, so they queue for agent review rather than auto-apply.
- Duplicate noise: finding ids must be stable and deduplicated by source artifact path plus task/record id.
</threat_model>

    - `agent-export-missing:{path}`
