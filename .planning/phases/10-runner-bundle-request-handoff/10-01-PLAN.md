<objective>
Make `run-task` prepare the patch bundle request handoff automatically.

Purpose: agents should be able to call `wikify run-task --id <id>` as the normal automation loop. If deterministic apply cannot proceed because the patch bundle is missing, the runner should still prepare the exact request artifact needed by an external agent, without prompting the user or hiding content generation inside the CLI.
</objective>

<data_flow>
1. `wikify run-task --id agent-task-1 --dry-run` previews proposal/lifecycle/request work and writes nothing.
2. `wikify run-task --id agent-task-1` creates or reuses the proposal.
3. If the patch bundle is missing, the runner writes `sorted/graph-patch-bundle-requests/<task-id>.json`.
4. The result returns `waiting_for_patch_bundle`, request artifact path, suggested bundle path, and `next_actions`.
5. An external agent reads the request and writes `sorted/graph-patch-bundles/<task-id>.json`.
6. The next `run-task` call applies the bundle and marks the task done.
</data_flow>
