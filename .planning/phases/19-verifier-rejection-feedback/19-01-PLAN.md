<objective>
Turn verifier rejection into durable task feedback.

Purpose: when automated maintenance stops because a verifier rejected a patch bundle, Wikify should leave the task in a clear `blocked` state with machine-readable feedback. This lets a later agent inspect, repair, and retry without asking the user to reconstruct what happened.
</objective>

<data_flow>
1. `run-task` reaches verifier gate after preflight.
2. Verifier returns `accepted: false`.
3. `verify_patch_bundle` writes `sorted/graph-patch-verifications/<task-id>.json` and raises `patch_bundle_verification_rejected`.
4. `run-task` catches that rejection, blocks the task through lifecycle, and attaches feedback details to the task and lifecycle event.
5. The error envelope keeps `verification_path`, `verdict`, `agent_tasks`, and `task_events` so automation can inspect next steps.
6. Existing `tasks --id <id> --retry` can move blocked work back to queued and clear stale block metadata.
</data_flow>
