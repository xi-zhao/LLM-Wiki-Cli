<objective>
Build a stable patch bundle request contract.

Purpose: when `run-task` reaches `waiting_for_patch_bundle`, agents should have a deterministic next command that packages all safe context needed to generate `wikify.patch-bundle.v1` outside the CLI. This keeps content generation explicit and auditable while reducing user interruptions.
</objective>

<data_flow>
1. `wikify bundle-request --task-id agent-task-1 --dry-run` reads the queue and proposal context, returns a request, and writes nothing.
2. `wikify bundle-request --task-id agent-task-1` writes or reuses `sorted/graph-patch-proposals/<task-id>.json`.
3. It writes `sorted/graph-patch-bundle-requests/<task-id>.json`.
4. The request includes proposal evidence, write scope, allowed `replace_text` contract, suggested bundle path, and target snapshots with SHA-256 hashes.
5. An external agent reads the request and writes `sorted/graph-patch-bundles/<task-id>.json`.
6. `wikify run-task --id agent-task-1` or `wikify apply` consumes the explicit bundle.
</data_flow>
