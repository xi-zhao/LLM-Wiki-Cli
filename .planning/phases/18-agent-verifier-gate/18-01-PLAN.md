<objective>
Build an explicit agent verifier gate before patch apply.

Purpose: let automated maintenance delegate review to another agent without interrupting the user, while preserving the deterministic apply/rollback boundary. The verifier reviews the proposal, bundle, request context, and deterministic preflight output, then accepts or rejects before any content mutation occurs.
</objective>

<data_flow>
1. Producer or existing bundle yields `sorted/graph-patch-bundles/<task-id>.json`.
2. Wikify runs deterministic `preflight_patch_bundle`.
3. If a verifier command/profile is explicitly configured, Wikify builds `wikify.patch-bundle-verification-request.v1`.
4. The verifier command receives that request on stdin and returns `wikify.patch-bundle-verdict.v1` JSON.
5. Wikify writes `sorted/graph-patch-verifications/<task-id>.json` with the verdict, preflight, and command evidence.
6. Accepted verdicts allow apply to continue.
7. Rejected verdicts return a structured non-retryable error before apply or lifecycle mark-done.
</data_flow>
