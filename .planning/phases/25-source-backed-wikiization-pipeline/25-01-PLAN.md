<objective>

<data_flow>
1. `wikify source add` records source identity in `.wikify/registry/sources.json`.
2. `wikify sync` records source item facts in `.wikify/sync/source-items.json` and queues changed/new work in `.wikify/queues/ingest-items.json`.
3. `wikify wikiize` loads the queue and source item index, applies selectors, and builds a dry-run or execution plan.
4. Local text/Markdown source items are read through the source item path and converted into conservative page drafts with title, summary, source refs, and bounded excerpts.
5. If explicit agent enrichment is requested, Wikify writes a `wikify.wikiization-request.v1` artifact, sends the request JSON to the agent command on stdin, captures a structured `wikify.wikiization-result.v1`, and still performs all final validation and writes itself.
6. Generated page metadata is rendered into Markdown front matter; generated page bodies are written under `wiki/pages/`.
7. Matching `wikify.wiki-page.v1` JSON object artifacts are written under `artifacts/objects/wiki_pages/`.
8. `artifacts/objects/object-index.json` is rebuilt from object artifacts.
9. `validate_workspace_objects(..., strict=True, write_report=True)` runs as the safety gate.
10. Completed entries are marked `completed`; blocked entries are marked `needs_review` or `failed` and get wikiization task records when agent follow-up is possible.
11. `.wikify/wikiization/last-wikiize.json` records counts, selected entries, generated object ids, paths, validation summary, task ids, and next actions.
</data_flow>

<threat_model>
## Threat Model

**Assets:**
- User-readable generated Markdown pages under `wiki/pages/`.
- Machine-readable wiki page objects under `artifacts/objects/wiki_pages/`.
- Source registry, source item index, and ingest queue integrity.
- User edits to generated pages.
- External agent request/result boundaries.

**Threats and mitigations:**
- Hidden provider behavior: `wikify wikiize` uses deterministic local generation by default and invokes external agents only when `--agent-command` or `--agent-profile` is explicit.
- Fake source-backed pages: remote-without-content, unsupported, unreadable, missing, skipped, or errored source items create tasks or failures instead of generated pages.
- Source traceability loss: generated wiki page objects and front matter require source refs with source/item ids and locator evidence; tests assert this.
- User edit overwrite: page updates require stored generated hash metadata matching current file content; drift creates review tasks.
- Partial or invalid objects: JSON writes use temp-file atomic replace; strict object validation runs after writes and prevents completed queue status on failure.
- Agent output path abuse: agent results are treated as content proposals only; Wikify chooses and validates final target paths inside generated wiki/object directories.
- Dependency creep: implementation remains stdlib-only and does not add YAML, Pydantic, JSON Schema, database, vector store, provider SDK, network fetch, or git-clone dependencies.
</threat_model>
