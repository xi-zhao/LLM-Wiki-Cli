<objective>
Build bounded maintenance loop automation.

Purpose: give users one low-interruption command that repeatedly refreshes graph maintenance and advances queued agent tasks until there is nothing actionable left or a configured safety bound is reached.
</objective>

<data_flow>
1. Caller runs `wikify maintain-loop --agent-profile --max-rounds 3 --task-budget 15`.
2. CLI resolves the explicit agent command or default profile sentinel before loop execution.
3. Round 1 calls `run_maintenance_workflow` with a per-round limit bounded by remaining budget.
4. `maintain-run` refreshes graph artifacts, selects tasks, invokes explicit agent production when available, applies deterministic bundles, and marks completed tasks done.
5. The loop records the full round result and updates aggregate counts.
6. The loop stops when no tasks are selected, a waiting/error state appears, dry-run preview is returned, task budget is exhausted, or max rounds are reached.
7. The final JSON envelope exposes status, stop reason, aggregate summary, per-round results, artifacts, and next actions.
</data_flow>
