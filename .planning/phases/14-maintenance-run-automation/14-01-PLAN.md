<objective>
Build `wikify maintain-run`, a low-interruption maintenance automation command.

Purpose: let an agent refresh graph findings and safely advance a bounded number of queued maintenance tasks in one command, while preserving the explicit external-agent boundary and deterministic patch application safety.
</objective>

<data_flow>
1. Caller runs `wikify maintain-run --policy balanced --limit 5 --agent-command "<explicit command>"`.
2. The command runs `run_maintenance(base, policy=..., dry_run=...)`.
3. Non-dry-run uses the newly written `sorted/graph-agent-tasks.json`.
4. Non-dry-run calls `run_agent_tasks` with status/action/id/limit and the explicit producer command if provided.
5. Dry-run uses the in-memory maintenance preview task queue to report which tasks would be selected, and does not call task producers or apply bundles.
6. The result exposes maintenance summary, task batch result or preview, artifacts, summary counts, and next actions.
</data_flow>
