<data_flow>
1. User or agent creates or receives object-bearing Markdown files and JSON object artifacts.
2. JSON object artifacts live under `artifacts/objects/`; the object index path is `artifacts/objects/object-index.json`.
3. Markdown pages can carry front matter at file start with required object metadata.
4. `wikify.markdown_index.scan_objects()` reads Markdown files, parses front matter metadata when present, and still returns path-compatible `WikiObject` records.
5. Graph extraction consumes `WikiObject` records exactly as before for path ids, while `GraphNode.to_dict()` can include `object_id` when metadata provides it.
6. `wikify.object_validation.validate_workspace_objects()` scans JSON artifacts and Markdown metadata, resolves ids, checks required fields, validates source refs against `.wikify/registry/sources.json` and `.wikify/sync/source-items.json` when present, and returns `wikify.object-validation.v1`.
7. `wikify validate` calls the validator with `--path` and `--strict` options and emits a stable JSON envelope.
8. Phase 25 consumes the object model and queue artifacts later; Phase 24 does not generate wiki pages from queue entries.
</data_flow>

<threat_model>
## Threat Model

**Assets:**
- User-readable wiki Markdown.
- Agent-readable JSON object artifacts.
- Source registry and source item integrity.
- Graph compatibility and maintenance task compatibility.
- Local workspace path boundaries.

**Threats and mitigations:**
- Parser overreach: `wikify/frontmatter.py` supports only documented scalar and JSON-flow values; invalid syntax becomes `object_frontmatter_invalid`.
- Content mutation by validation: validator reads Markdown/JSON and writes only optional validation reports; it does not edit wiki pages.
- Path traversal from `wikify validate --path`: focused paths must resolve under the workspace/base directory unless the user explicitly passes an absolute path inside that base; outside paths return an exit-code-2 validation error.
- Source truth fork: source/source item adapters are read-only views over Phase 22/23 artifacts; no new source registry is written.
- Graph regression: graph node ids remain `relative_path`; object ids are additive metadata.
- Dependency creep: verification greps for `pydantic`, `jsonschema`, `yaml`, `ruamel`, `sqlite3`, `requests`, `urllib.request`, and `subprocess` in the new object/front matter/validation modules.
- Large source copying in citations: citation objects store ids, locators, spans, confidence, and optional bounded snippets only; they do not embed full raw source documents.
</threat_model>
