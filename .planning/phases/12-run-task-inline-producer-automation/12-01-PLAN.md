<objective>
Build one-command task automation by letting `run-task` explicitly call the Phase 11 bundle producer when the caller provides `--agent-command`.

Purpose: reduce external orchestration from `run-task -> produce-bundle -> run-task` to one command while preserving the hard boundary that Wikify does not choose providers, models, keys, or retry policy.
</objective>

<data_flow>
1. Caller runs `wikify run-task --id agent-task-1 --agent-command "<explicit command>"`.
2. Runner creates or reuses the proposal.
3. Runner marks queued tasks as proposed in non-dry-run.
4. If the bundle is missing, runner writes `sorted/graph-patch-bundle-requests/<task-id>.json`.
5. Runner invokes `produce_patch_bundle` with that request and the explicit command.
6. Producer writes or accepts `sorted/graph-patch-bundles/<task-id>.json` and preflights it.
7. Runner applies the bundle through deterministic apply and marks the task done.
</data_flow>
