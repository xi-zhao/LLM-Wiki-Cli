<data_flow>
1. User or agent runs `wikify views`.
2. The command discovers the workspace through `discover_base()` and loads the workspace manifest plus source registry.
3. The view module loads `artifacts/objects/object-index.json` and object JSON files under `artifacts/objects/`.
4. The view module loads optional source/control artifacts: `.wikify/sync/source-items.json`, `.wikify/queues/ingest-items.json`, `.wikify/queues/wikiization-tasks.json`, `artifacts/objects/validation.json`, and graph artifacts under `graph/`.
5. `--dry-run` builds view plans and returns planned Markdown/HTML paths without writing any files.
6. Non-dry-run validates object artifacts before rendering; hard validation errors return a structured `views_validation_failed` error.
7. The renderer builds Markdown pages for home, pages, sources, collections, timeline, graph, and review.
8. The write path checks `.wikify/views/view-manifest.json`; safe generated files are overwritten, drifted Markdown views are preserved and converted into `.wikify/queues/view-tasks.json` entries.
9. If HTML output is enabled, the static renderer converts generated Markdown views into escaped local HTML under `views/site/` and writes `views/site/assets/style.css`.
10. `.wikify/views/view-manifest.json` records generated paths and hashes. `.wikify/views/last-views.json` records counts, conflicts, artifacts, warnings, validation summary, and next actions.
11. `wikify views` returns a `views` JSON envelope with completion metadata.
</data_flow>

<threat_model>
## Threat Model

**Assets:**
- Human-facing generated Markdown views under `views/`.
- Local static HTML output under `views/site/`.
- Machine-readable object artifacts under `artifacts/objects/`.
- Source registry, source item index, queue artifacts, validation records, and graph artifacts.
- User edits to visible generated view Markdown.

**Threats and mitigations:**
- Hidden pipeline execution: `wikify views` only renders existing artifacts; tests assert it does not call sync, wikiize, graph, agents, providers, network, repository, or background behavior.
- Source truth drift: views read object/source/control artifacts and never create a separate human-only store.
- Fake human summaries: views display object summaries, source refs, citations, and task evidence; they do not infer facts from raw source text.
- User edit overwrite: Markdown views are overwritten only when `.wikify/views/view-manifest.json` proves current content still matches the prior generated hash; drift creates view tasks.
- Unsafe HTML: renderer escapes text through stdlib `html.escape`, emits relative links only, and does not include external scripts, telemetry, or CDN assets.
- Validation blindness: hard object validation errors block non-dry-run generation; warnings appear in review views.
- Dependency creep: implementation remains stdlib-only and avoids Markdown/YAML/template/JS/server/vector/provider dependencies.
- Path traversal: all generated paths are fixed relative paths under `views/`, `views/site/`, `.wikify/views/`, or `.wikify/queues/`; no object title or source locator can choose output directories.
</threat_model>
