<data_flow>
1. User runs `wikify sync` or `wikify sync --source src_<id>`.
2. Wikify resolves the workspace through existing `discover_base()` behavior.
3. Wikify loads `wikify.json` and `.wikify/registry/sources.json`.
4. Wikify loads the previous `.wikify/sync/source-items.json` if it exists.
5. Wikify discovers source items for each selected source.
6. Wikify computes deterministic fingerprints and compares them to the previous source item index.
7. Wikify classifies items as `new`, `changed`, `unchanged`, `missing`, `skipped`, or `errored`.
8. Non-dry-run sync writes `.wikify/sync/source-items.json`, `.wikify/sync/last-sync.json`, `.wikify/queues/ingest-items.json`, and source-level sync metadata in `.wikify/registry/sources.json`.
9. Dry-run returns the same planned result shape but writes nothing.
10. Later phases consume `.wikify/queues/ingest-items.json`; Phase 23 does not consume the queue.
</data_flow>

<threat_model>
## Threat Model

**Assets:**
- Source registry integrity.
- Source item index integrity.
- Ingest queue correctness.
- Local file paths and workspace boundaries.
- Agent-facing JSON artifacts consumed by future automation.

**Threats and mitigations:**
- Hidden network/provider execution: `wikify sync` must not import `requests`, use `urllib.request`, call subprocesses, invoke `git`, fetch URLs, clone repositories, run existing ingest scripts, or call providers.
- Path traversal or out-of-workspace writes: sync writes only `.wikify/sync/source-items.json`, `.wikify/sync/last-sync.json`, `.wikify/queues/ingest-items.json`, and `.wikify/registry/sources.json` under the workspace root.
- Expensive scans or hash blowups: directory scans use deterministic ignore rules, regular-file-only traversal, and a bounded SHA-256 size cap.
- Queue duplication: queue entries are keyed by deterministic item id and upserted, not appended blindly.
- Data loss on write failure: write derived sync artifacts through temp-file plus atomic replace before updating registry metadata.
- Bad source aborting all sync: per-source and per-item errors are recorded, and unrelated sources continue.
</threat_model>
