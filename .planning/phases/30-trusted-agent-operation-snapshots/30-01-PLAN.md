**Status:** Complete

## Acceptance Checks

- `begin` writes `.wikify/trusted-operations/<operation-id>.json` with before snapshots and reason.
- `complete` records after snapshots and makes rollback available.
- `rollback --dry-run` validates without changing files.
- `rollback` restores modified/deleted files and removes newly created files when hashes match.
- Rollback refuses drifted content with a structured error.
- CLI returns structured JSON envelopes for trusted-op commands.
- Full tests, compile check, and diff check pass.

## Notes

This phase adds explicit recovery primitives. It does not try to automatically monitor every filesystem write.
