<data_flow>
1. User or agent runs `wikify agent export` or `wikify agent export --dry-run`.
2. The CLI resolves the workspace with `discover_base()` and calls `run_agent_export(base, dry_run=..., include_full=..., max_full_chars=..., max_page_chars=...)`.
3. The agent module loads `wikify.json`, source registry, optional source item index, optional validation report, optional view manifest/report, optional legacy `graph/graph.json`, `artifacts/objects/object-index.json`, and object JSON documents under `artifacts/objects/`.
4. Dry-run builds the planned artifact list and summary without writing `llms.txt`, `llms-full.txt`, `artifacts/agent/`, or `.wikify/agent/`.
5. Non-dry-run runs strict object validation on `artifacts/objects/`; hard errors raise `AgentInterfaceError(code='agent_validation_failed')`.
6. The module builds deterministic `page-index.json`, `citation-index.json`, `related-index.json`, and object-model-first `graph.json`.
7. The module renders compact `llms.txt` and richer `llms-full.txt` from page/index artifacts and generated page bodies, never raw source files.
8. The module writes visible artifacts under workspace root and `artifacts/agent/`, writes `.wikify/agent/last-agent-export.json`, and returns a `wikify.agent-export.v1` result.
9. The CLI wraps success with `envelope_ok('agent.export', result)` and errors with stable exit-code-2 envelopes.
</data_flow>

<threat_model>
## Threat Model

**Assets:**
- Root `llms.txt` and `llms-full.txt`.
- Agent indexes under `artifacts/agent/`.
- Source/object/wiki/view/graph artifacts consumed by the export.
- Stable JSON envelopes and exit codes consumed by agents.

**Threats and mitigations:**
- Hidden pipeline execution: `wikify agent export` reads existing artifacts only and tests assert it does not call sync, wikiize, views, graph, providers, agents, network, repository commands, or watchers.
- Citation fabrication: citation index entries are derived only from explicit `citation` objects and page-level `source_refs`; entries mark evidence strength.
- Invalid object artifacts: non-dry-run export runs strict object validation and returns `agent_validation_failed` with validation details on hard errors.
- Silent missing data: optional missing graph/view/citation artifacts produce warnings and reduced indexes, not fake entries.
- Opaque related scores: related index stores signals and weights, not only numeric score.
- Silent truncation: `llms-full.txt` report records `max_full_chars`, `max_page_chars`, included chars, omitted page count, and truncation flags.
- Path traversal: output paths are fixed constants under root, `artifacts/agent/`, and `.wikify/agent/`; object titles never choose output paths.
- Dependency creep: implementation remains stdlib-only and avoids Markdown/YAML/template/vector/provider dependencies.
</threat_model>
