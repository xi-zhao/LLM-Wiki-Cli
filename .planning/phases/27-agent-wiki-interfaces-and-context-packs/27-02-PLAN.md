<data_flow>
1. User or agent runs `wikify agent context <query>`, `wikify agent cite <query-or-object-id>`, or `wikify agent related <object-id-or-query>`.
2. The CLI resolves the workspace with `discover_base()` and calls focused functions in `wikify.agent`.
3. Query functions load the same object/source/wiki/view/graph snapshot used by `agent export`. They may read existing `artifacts/agent/` indexes for speed, but must be able to compute results from the snapshot without writing those indexes.
4. `agent cite` returns explicit citation evidence first, page source-ref fallback evidence second, and empty evidence plus next actions when nothing supports the query.
5. `agent related` returns ranked related objects/pages with score, confidence, and signal maps.
6. `agent context --dry-run` returns the would-select set, budget accounting, rationale, citations/source refs, and planned pack paths without writing files.
7. Non-dry-run `agent context` validates object artifacts, selects deterministic source-backed page excerpts, writes `artifacts/agent/context-packs/<pack-id>.json`, writes a matching `wikify.context-pack.v1` object under `artifacts/objects/context_packs/`, updates `.wikify/agent/context-pack-manifest.json`, updates object index with the context pack object, and returns a `wikify.context-pack.v1` result.
</data_flow>

<threat_model>
## Threat Model

**Assets:**
- Context pack JSON and context pack object artifacts.
- Citation evidence returned to agents.
- Related query signals and scores.
- Source-backed wiki page excerpts included in context packs.

**Threats and mitigations:**
- Fabricated claim support: `agent cite` only returns explicit citation objects and page-level source refs, with evidence type clearly marked.
- Context budget explosion: context packs enforce `--max-chars`, `--max-pages`, per-item excerpt lengths, included char count, omitted count, and truncation flags.
- Hidden semantic selection: context selection uses lexical match, object ids, titles, summaries, source refs, citations, links, graph edges, and related signals only.
- Raw source rereads: context packs include generated page excerpts and object metadata; they do not reopen raw source files.
- Stale or missing indexes: query commands can compute from current object artifacts and report warnings/next actions rather than silently returning fake completeness.
- Invalid writes: non-dry-run context pack writes validate object artifacts first and refresh object index with the pack object.
- Path traversal: pack ids are generated from stable digests; query text never chooses output paths.
- Unbounded writes: `agent cite` and `agent related` are read-only; only `agent context` writes pack artifacts, and dry-run writes nothing.
</threat_model>
